Boggle Server WebSocket API 1.0.0 documentation

Real-time WebSocket API for multiplayer and solo Boggle gameplay. The WebSocket server shares a port with the HTTP server. All client messages (except the raw connection-ID frame) are JSON.

Servers

production Server

  • URL: wss://{host}
  • Protocol: wss

If the server has an API token configured, include it as a query parameter: wss://host?token=. The token is validated at connection time. boardHistoryCount is the only event that works on unauthenticated connections.

URL Variables

Name Description Default value Allowed values
host Server hostname and optional port (e.g. example.com or localhost:443) None Any

insecure Server

  • URL: ws://{host}
  • Protocol: ws

Used when SKIP_TLS=1 is set in the server .env

URL Variables

Name Description Default value Allowed values
host Server hostname and optional port None Any
